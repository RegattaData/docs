---
title: SQL Syntax Conventions
---

e following conventions are used in the synopsis of a command:

1. brackets (**[ ]**) indicate optional parts.
1. Braces (**{ }**) and vertical lines (**|**) indicate that you must choose one alternative.
1. Dots (**...**) mean that the preceding element can be repeated.
1. **lowercase bold consolas** font indicates a placeholder which should be replaced with an actual value (e.g: **table\_name**, **parameter\_name**, etc).
1. `UPPERCASE` `COURIER` `NEW` font indicates SQL language literals (e.g: `SELECT`)
1. lowercase courier new font indicates parameter names (e.g: devices)

All other symbols, including parentheses, should be taken literally.
